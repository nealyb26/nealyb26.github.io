Embedded Flight Phase Detection System
View Code Technical Report

Overview

For the last 2 years I have been a member of the electronics team for the Vanderbilt Aerospace Design Lab. For the 2026 Student Launch Competition, I worked on the flight logic of our payload electronics system on our rocket, the Endurance. The state detection is handled by an embedded system that uses inertial measurement unit (IMU) data to detect different phases of flight in real-time. The system implements a robust finite-state machine to identify launch, boost, coast, apogee, and recovery phases. These states are used to dictate action events of the launch system throughout flight, such as parachute detachment and soil collection system activation.

Developed for applications in model rocketry and small unmanned aircraft, the system provides critical flight phase information that can be used for automated recovery system deployment, data logging, and safety monitoring.

Technical Implementation

The core system runs on an STM32 Nucleo board with a high-precision 9-axis IMU providing accelerometer, gyroscope, and magnetometer data at 100Hz. A Kalman filter processes the raw sensor data to estimate vehicle orientation and acceleration, filtering out noise and sensor drift.

The finite-state machine analyzes filtered sensor data using configurable thresholds and timing windows to detect state transitions. Each flight phase has specific acceleration and rotation rate signatures that the algorithm recognizes, with hysteresis built in to prevent false state changes due to turbulence or sensor noise.

FreeRTOS provides real-time scheduling for sensor reading, data processing, and state machine execution, ensuring deterministic timing critical for flight safety applications. The system includes comprehensive unit tests and desktop simulation capabilities for validation without flight hardware.

Key Features

  • Real-time flight phase detection with sub-100ms latency
  • Robust finite-state machine with noise immunity
  • Kalman filtering for sensor fusion and noise reduction
  • Configurable thresholds for different vehicle types
  • Desktop simulation environment for testing
  • Comprehensive unit test suite
  • Low power consumption for extended flight operations
  • Serial output for data logging and telemetry

Testing and Validation

The system was validated using a desktop test harness that simulates various flight profiles through controlled motion patterns. This allowed for repeatable testing of edge cases and boundary conditions without requiring actual flight tests.

Unit tests cover individual components including sensor drivers, filtering algorithms, and state machine logic. Integration tests verify end-to-end functionality using recorded flight data from previous missions, ensuring the system correctly identifies flight phases in realistic scenarios.

Field testing on a Nucleo board confirmed the system's reliability and accuracy, with successful state detection across multiple simulated flight profiles and environmental conditions.

Applications

The flight state detection system has applications in model rocketry for automated parachute deployment, small UAV flight monitoring, and educational aerospace projects. The modular design allows easy integration into existing flight systems and adaptation for different vehicle characteristics.
